# Shot Spec: Troubleshooting Login and Access

| # | Shot | UI state to set up | Cursor | Filename | Width |
|---|---|---|---|---|---|
| 1 | Sign-in page (logged out) | Logged out state, on the sign-in page. Show the full split-layout sign-in screen with email field, password field, eye show/hide toggle, right-aligned "Forgot Password?" link, dark "Sign In" button, and "Don't have an account? Sign Up" line. | arrow on the "Sign In" button | `signin-full-page.png` | ~840 CSS |
| 2 | Calendars tab (connected state) | Navigate to **Settings** (profile picture → **Settings**) → **Calendar & Scheduling** under Account group → **Calendars** tab. A calendar should be shown with the green "Connected" badge. Include the "+ Connect calendar" button at the top right of the card and the left settings sidebar showing the Account group context. Sanitize any customer email addresses in the calendar row. | arrow on the "+ Connect calendar" button | `calendars-connected.png` | ~840 CSS |
| 3 | Calendars tab (disconnected state) — **optional** | Navigate to **Settings** → **Calendar & Scheduling** → **Calendars** tab in a state where no calendar is connected. Show the "Connect Google Calendar" and "Connect Office 365 Calendar" entry buttons. Sanitize any email addresses. **Only capture if a throwaway account is available** — do not disconnect the shared account's calendar. If unavailable, rely on Shot 2 alone; the "+ Connect calendar" button is sufficient. | arrow on one of the connect entry buttons (e.g., "Connect Google Calendar") | `calendars-disconnected.png` | ~840 CSS |
